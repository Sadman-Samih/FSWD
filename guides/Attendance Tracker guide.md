## 1. How It Works

Your app is a single web page with three parts:

| Part | What it does |
|---|---|
| **HTML** | Defines the structure: title, month/year dropdowns, and a table |
| **CSS** | Makes it look good: colors, fonts, sticky name column, round buttons |
| **JavaScript** | Connects to Supabase, fetches employees and attendance logs, draws the grid, handles clicks |

**Database flow:**

1. App loads → JS asks Supabase: "Give me all employees" and "Give me logs for this month"
2. JS draws a table: rows = employees, columns = days of the month
3. You click a cell → JS tells Supabase: "Insert/update/delete this record"
4. JS redraws the table to show changes

**Supabase** is a free cloud database. It stores two tables:
- `employees` — just a list of names (X, Poli, Sharif)
- `attendance_logs` — one row per click: employee_id, date, status (Present), created_at (timestamp of when it was recorded)

## 2. Supabase Setup

### 2.1 Create a Supabase account

1. Go to [https://supabase.com](https://supabase.com) and click **Start your project**
2. Sign up with GitHub or email
3. Once logged in, click **New project**
4. Choose an organization (or create one), give it a name (e.g. "Attendance Tracker")
5. Set a **Database Password** — save it somewhere safe
6. Choose a region close to you
7. Click **Create new project** (takes ~1 minute)

### 2.2 Create the tables

Once the project is ready:

1. In the left sidebar, click **SQL Editor**
2. Click **New query**
3. Paste this and click **Run**:

```sql
drop table if exists attendance_logs;
drop table if exists employees;

create table employees (
  id bigint generated by default as identity primary key,
  name text not null
);

create table attendance_logs (
  id bigint generated by default as identity primary key,
  employee_id bigint references employees(id) on delete cascade,
  date date not null,
  status text not null,
  constraint unique_employee_date unique (employee_id, date)
);

insert into employees (name) values
  ('X'),
  ('Poli'),
  ('Sharif');
```

**What this does:**
- Deletes old tables if they exist (clean start)
- Creates `employees` with an auto-incrementing `id` and a `name`
- Creates `attendance_logs` with an id, employee reference, date, and status
- The `unique_employee_date` constraint prevents duplicate entries for the same person on the same day
- Inserts your 3 employees

### 2.3 Disable Row Level Security (RLS)

Since we're not using logins, we need to let anyone read/write:

1. In the left sidebar, click **Table Editor**
2. Click the `employees` table
3. Find **RLS Enabled** toggle and turn it **OFF**
4. Click `attendance_logs` table and turn RLS **OFF** too

### 2.4 Get your API keys

1. In the left sidebar, click the **Settings** gear icon (bottom)
2. Click **API**
3. Copy the **Project URL** (looks like `https://xxxxxxx.supabase.co`)
4. Copy the **anon public** key (starts with `eyJ...`)
5. Keep these handy — you'll put them in your code

> **IMPORTANT:** Use the **anon** key, not the `service_role` key. The `service_role` key is a secret admin key — never put it in frontend code. The anon key is safe for public use when RLS is disabled.

## 6. js/supabase.js — The Database Connection

This file initializes Supabase with your project credentials.

```javascript
'use strict';

const SUPABASE_URL = 'https://YOUR_PROJECT_ID.supabase.co';
const SUPABASE_ANON_KEY = 'eyJ...your-anon-key-here...';

window.supabase = window.supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);
```

**Replace** `YOUR_PROJECT_ID` and the key with what you copied from Supabase → Settings → API.

**Why `window.supabase = …` instead of `const supabase = …`?**
The Supabase CDN library already defines `window.supabase` as a read-only property. If we try `const supabase = …`, JavaScript throws an error because you can't redeclare a non-configurable global. By assigning directly to `window.supabase`, we simply replace it with our client instance. Then in `app.js`, writing just `supabase` works because JavaScript looks up `supabase` on `window` automatically.

## 7. js/app.js — The Logic

This is the brain of the app. It's organized as a single `App` object with methods.

```javascript
'use strict';

const App = {
  // Track which month/year is selected
  state: {
    monthIdx: new Date().getMonth(),
    year: new Date().getFullYear(),
  },

  // Runs once when the page loads
  async init() {
    const monthSel = document.getElementById('month-select');
    const yearSel = document.getElementById('year-select');

    // Set dropdowns to current month/year
    monthSel.value = this.state.monthIdx;
    yearSel.value = this.state.year;

    // Re-render when dropdown changes
    monthSel.addEventListener('change', () => {
      this.state.monthIdx = parseInt(monthSel.value);
      this.renderGrid();
    });

    yearSel.addEventListener('change', () => {
      this.state.year = parseInt(yearSel.value);
      this.renderGrid();
    });

    // Event delegation: one listener for ALL cell clicks
    document.getElementById('tbody').addEventListener('click', (e) => {
      const btn = e.target.closest('.cell-btn');
      if (!btn) return;
      this.toggleStatus(btn);
    });

    // Draw the table for the first time
    await this.renderGrid();
  },

  // Fetch data and rebuild the table
  async renderGrid() {
    const { monthIdx, year } = this.state;
    const monthName = document.getElementById('month-select').options[monthIdx].text;
    document.getElementById('sheet-title').textContent =
      `Attendance Sheet for ${monthName} - ${year}`;

    // Number of days in the selected month
    const totalDays = new Date(year, monthIdx + 1, 0).getDate();

    // Fetch employees AND logs at the same time (parallel)
    const [employees, logs] = await Promise.all([
      this.fetchEmployees(),
      this.fetchLogs(year, monthIdx),
    ]);

    // Build a lookup map: logsMap[employeeId][dateString] = 'Present'
    const logsMap = {};
    for (const log of logs) {
      if (!logsMap[log.employee_id]) logsMap[log.employee_id] = {};
      logsMap[log.employee_id][log.date] = log.status;
    }

    // --- Draw table header ---
    const thead = document.getElementById('table-head');
    let headerHtml = '<tr><th class="col-name">Student Name</th>';
    for (let d = 1; d <= totalDays; d++) {
      headerHtml += `<th>${d}</th>`;
    }
    headerHtml += '<th class="col-present-count">Present Count</th></tr>';
    thead.innerHTML = headerHtml;

    // --- Draw table body ---
    const tbody = document.getElementById('tbody');
    let bodyHtml = '';

    for (const emp of employees) {
      let presentCount = 0;
      let rowHtml = `<tr><td class="col-name">${emp.name}</td>`;

      for (let d = 1; d <= totalDays; d++) {
        // Format date as YYYY-MM-DD (e.g. 2026-06-15)
        const dateStr = `${year}-${String(monthIdx + 1).padStart(2, '0')}-${String(d).padStart(2, '0')}`;
        const status = logsMap[emp.id]?.[dateStr] ?? null;

        let cls = 'cell-btn';
        let icon = '';
        if (status === 'Present') {
          cls += ' cell-present';
          icon = '✓';
          presentCount++;
        }

        // Store data in data-* attributes for use on click
        rowHtml += `<td><button class="${cls}" data-employee="${emp.id}" data-date="${dateStr}" data-status="${status ?? ''}">${icon}</button></td>`;
      }

      rowHtml += `<td class="col-present-count">${presentCount}</td></tr>`;
      bodyHtml += rowHtml;
    }

    // If no employees in database, show a friendly message
    tbody.innerHTML = bodyHtml ||
      '<tr><td colspan="99" style="text-align:center;padding:2rem;color:#999;">No employees found. Add some in Supabase.</td></tr>';
  },

  // Handle a cell click: toggle between empty and Present
  async toggleStatus(btn) {
    const employeeId = parseInt(btn.dataset.employee);
    const dateStr = btn.dataset.date;
    const currentStatus = btn.dataset.status;
    const newStatus = currentStatus === 'Present' ? '' : 'Present';

    if (newStatus === '') {
      await supabase
        .from('attendance_logs')
        .delete()
        .eq('employee_id', employeeId)
        .eq('date', dateStr);
    } else {
      await supabase
        .from('attendance_logs')
        .upsert({ employee_id: employeeId, date: dateStr, status: newStatus });
    }

    // Redraw to reflect changes
    await this.renderGrid();
  },

  // Get all employees from Supabase
  async fetchEmployees() {
    const { data, error } = await supabase
      .from('employees')
      .select('*')
      .order('name');
    if (error) console.error('fetchEmployees error:', error);
    return data || [];
  },

  // Get attendance logs for a specific month
  async fetchLogs(year, monthIdx) {
    const month = String(monthIdx + 1).padStart(2, '0');
    const start = `${year}-${month}-01`;
    const end = `${year}-${month}-${new Date(year, monthIdx + 1, 0).getDate()}`;

    const { data, error } = await supabase
      .from('attendance_logs')
      .select('*')
      .gte('date', start)
      .lte('date', end);
    if (error) console.error('fetchLogs error:', error);
    return data || [];
  },
};

// Start the app when the page is ready
document.addEventListener('DOMContentLoaded', () => App.init());
```

### How the JavaScript works — line by line

**The `state` object** holds the currently selected month (0-11) and year. It starts with the real current date.

**`init()`** is called when the page loads. It:
1. Sets the dropdown values to the current month/year
2. Listens for dropdown changes to re-render
3. Sets up one click listener on the table body (event delegation — more on this below)
4. Calls `renderGrid()` to draw the table

**`renderGrid()`** is the main function:
1. Reads the selected month/year
2. Updates the title (e.g. "Attendance Sheet for June - 2026")
3. Calculates how many days are in the month — the trick `new Date(year, monthIdx + 1, 0).getDate()` gives you the last day number (28-31)
4. Fetches employees and logs in parallel with `Promise.all()` — both requests happen at the same time instead of waiting for one then the other
5. Builds a `logsMap` — a nested object that lets us quickly look up a status: `logsMap[employeeId][dateString]`
6. Constructs HTML strings for the header (day numbers) and body (employee rows with buttons)
7. Injects the HTML into the page

**`toggleStatus(btn)`**:
1. Reads `data-*` attributes from the clicked button
2. Toggles: empty → Present (✓) → empty
3. If deleting, calls `supabase.from('attendance_logs').delete()`
4. If inserting/updating, calls `supabase.from('attendance_logs').upsert()`
5. Redraws the grid

**Event delegation:** Instead of attaching a click handler to every single button (there could be hundreds), we attach ONE handler to the `<tbody>`. When a button is clicked, the event "bubbles up" through the DOM. The handler checks if the click target (or any parent) has the class `cell-btn`. This is more efficient and simpler.

**Supabase queries explained:**
- `.select('*')` — get all columns
- `.order('name')` — sort alphabetically by name
- `.gte('date', start)` — greater than or equal to start date
- `.lte('date', end)` — less than or equal to end date
- `.eq('employee_id', id)` — filter where column equals value
- `.upsert({...})` — insert if new, update if exists (based on the `unique_employee_date` constraint)
- `.delete()` — delete matching rows

## 8. PWA — Install Like a Mobile App

Your site can be installed on a phone home screen just like a native app. This is called a **Progressive Web App (PWA)**.

### What we set up

**`manifest.json`** — tells the browser how to display your app:
```json
{
  "name": "Attendees",
  "short_name": "Attendees",
  "start_url": ".",
  "display": "standalone",
  "background_color": "#e9ecef",
  "theme_color": "#2b6cb0",
  "icons": [
    { "src": "icon.svg", "sizes": "any", "type": "image/svg+xml", "purpose": "any maskable" }
  ]
}
```

**`sw.js`** (service worker) — caches all static files on first visit so the app loads offline:
- First visit: downloads and caches HTML, CSS, JS, icon
- Subsequent visits: serves from cache instantly, even without internet
- Supabase data still needs internet (but the page itself loads)

**`icon.svg`** — a simple blue square with a white checkmark used for the home screen icon.

### How to install

| Device | Steps |
|---|---|
| **Android (Chrome)** | Visit the site → tap the banner "Add to Home Screen" or menu → Add to Home Screen |
| **iPhone (Safari)** | Tap the Share icon → scroll down → Add to Home Screen |
| **Desktop (Chrome/Edge)** | Click the install icon in the address bar |

The app opens without browser tabs/URL bar, just like a real app.

### Service worker code explained

```javascript
const CACHE = 'attendees-v1';
const STATIC = ['index.html', 'css/style.css', 'js/supabase.js', 'js/app.js', ...];

// On install: download all static files into cache
self.addEventListener('install', (e) => {
  e.waitUntil(caches.open(CACHE).then((cache) => cache.addAll(STATIC)));
});

// On fetch: serve from cache if available, otherwise fetch from network
self.addEventListener('fetch', (e) => {
  e.respondWith(caches.match(e.request).then((r) => r || fetch(e.request)));
});
```

The registration script in `index.html` is just:
```html
<script>
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('sw.js');
  }
</script>
```

## 9. Security Notes

Since there's no login, your only protection is keeping the URL private:

- **Don't post the URL** on social media or public forums
- **Use an anon key**, never the `service_role` key
- **RLS must be disabled** for both tables (or the app won't be able to read/write)
- If you need real security later, build the login version (see the login guide)

**What a bad actor could do:** Anyone with the URL could delete attendance data via the browser's developer tools. For a low-risk board attendance sheet among trusted people, this is acceptable. For anything sensitive, add authentication.

## 10. Troubleshooting

### "redeclaration of non-configurable global property supabase"

**Cause:** `const supabase = window.supabase.createClient(...)` conflicts with the Supabase CDN library.

**Fix:** Use `window.supabase = window.supabase.createClient(...)` instead.

### "supabase.from is not a function"

**Cause:** The database client wasn't created properly — usually the same issue as above.

**Fix:** Check `supabase.js` and verify the fix above.

### "Failed to fetch" or network errors

**Causes:**
- Supabase URL is wrong (check for typos)
- RLS is still enabled (turn it off in Table Editor)
- You used the `service_role` key by mistake
- You opened the file directly (use Live Server)
- No internet connection

### Blank page, no errors

**Cause:** The `employees` table is empty.

**Fix:** Run the SQL script to insert employees. The app shows a message "No employees found" if the table exists but is empty — if even that message doesn't appear, check the browser console (F12) for errors.

### Table shows but buttons don't work when clicked

**Causes:**
- `supabase.js` is loading after `app.js` (check script order in HTML)
- Supabase credentials are wrong
- RLS is enabled

### How to check for errors

1. Press **F12** to open Developer Tools
2. Click the **Console** tab
3. Look for red text — it tells you exactly what's wrong and which file/line

## Quick Reference: Supabase SQL Commands

```sql
-- View all employees
select * from employees;

-- View all attendance logs
select * from attendance_logs;

-- Add a new employee
insert into employees (name) values ('New Name');

-- Delete an employee (and their logs)
delete from employees where name = 'Old Name';

-- Delete all logs for a specific month
delete from attendance_logs
where date >= '2026-06-01' and date <= '2026-06-30';

-- See attendance summary
select e.name, count(al.id) as present_days
from employees e
left join attendance_logs al on al.employee_id = e.id and al.status = 'Present'
group by e.name
order by e.name;
```
